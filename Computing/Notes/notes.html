B. 2.1.2 Computing Hardware
1. The CPU
a) What is the CPU
(1) The CPU processes all data in three main parts
i) Control unit: control the operation of the CPU
ii) ALU: carries out mathematical and logical computations
iii) Registers: Memory locations in the CPU that store data and instructions
b) How does it work
(1) It runs on a Fetch Decode Execute cycle
(2) Fetch: instructions are fetched from the memory
(3) Decode: the Control unit finds out what the instructions mean
(4) Execute: the ALU executes the instructions and stores it in the register
c) Performance
(1) The clock speed is the speed at which the CUP runs the FDE cycle
(2) It is measured in Hertz. 1Hz - 1 cycle per second. 3 GHz - 3 Billion cycles per second
(3) The number of cores means how many processors are in the CPU. 2 cores means 2 instructions can be carried out simultaneously
(4) A larger cache size means more data can be stored on the CPU its self, meaning quicker access to the data.
2. Binary
a) Why is binary used
(1) Binary is a base-2 system, where all decimal numbers can be represented with 2 digits
(2) It is much faster to determine if a component is on or off (digital) or if it has a 0-9 property (analogue)
b) Logic gates
(1) Logic gates are used to take two binary inputs and calculate a result
(2) AND gate
i) Both inputs must be True for output to be True
ii) 00:0 10:0 01:0 11:1
(3) OR gate
i) Either input must be True for output to be True
ii) 00:0 01:1 10:1 11:1
(4) NOT gate
i) Input must be False for output to be True
ii) 0:1 1:0
3. Memory
a) RAM and ROM
(1) RAM - Random access Memory
(2) Volatile so looses data when not powered
(3) Read&Write access
(4) ROM - Read Only Memory
(5) Non-Volatile so data remains when not powered
(6) Read Only access (write access is very slow and not user accessible)
b) Use of ROM
(1) ROM is sued to store the BIOS
(2) The Basic I/O System is what first turns the computer on and initialises the OS
(3) If this is lost or corrupted, the computer will not work
c) Use of RAM
(1) The RAM is the temporary storage for data that is being used by the CPU
(2) It is much quicker than retrieving it from secondary storage
(3) This ensures that the CPU has a constant data flow to make it faster
(4) If there is more RAM in a system, it means more data can be stored ready for the CPU, meaning it can multitask more programs, reducing the use of vRAM, and making the system faster
d) vRAM
(1) When the RAM is full, data that isnâ€™t currently being used is sent to the secondary storageâ€™s vRAM section
(2) This means that it is immediately ready for use, but is slower than RAM
e) Cache
(1) Cache is the last stage of storage before the CPU
(2) The next data to be used by the CPU is moved to L1, L2, or L3 cache.
(3) The cache also stores information that has been used many times, such as keyboard input instructions.
f) Flash
(1) Flash memory us a type of solid state memory, similar to RAM
(2) It is however non-volatile, so can be used as secondary storage
(3) It has no movie parts so is quicker, safer, and faster, but more expensive.

5. Secondary Storage
a) Secondary storage is the additional storage to store data and programs. This is needed so that data is not lost when not powered. It is therefore Non-Volatile Read&Write
b) Common types
(1) Optical: CDs or DVDs. A laser scans for color change in the surface (similar to vinyl)
(2) Magnetic: mHDD or Floppy disks. Holds data in a magnetic polarity on a spinning disk
(3) Solid state: SSD/Flash, SD cards. Slower than RAM, faster than magnetic. It has no moving parts, but constant read/write can cause deterioration
c) What to use
(1) There are a number of factors to consider when deciding what media to use
(2) Capacity: Optical - Small. Magnetic - Large. SSD - medium
(3) Portability: Optical and SSD - yes. Magnetic - not when running
(4) Reliability: Optical - poor. Magnetic - Best. SSD - Medium
(5) Durability: Optical - prone to scratches. Magnetic - delicate disk and spindle. SSD - very durable
(6) Speed: Optical - Slowest. Magnetic - Medium. SSD - Fastest

F. 2.1.6 Communications and Networking
1. Networks
a) LANs
(1) Local Area Networks are set up between computers with NICs
(2) They allow resources to be shared between users
(3) NICs are Network Interface Cards
b) Hardware
(1) NICs are always required
(2) Hub/Switch are used to sort and send the data packets to the right computer
(3) WAPs allow mobile users to connect wirelessly
c) Structures
(1) P2P
i) This is a Peer to Peer network, where there is no one centralised server, and all computers share information
(2) Client-Server
i) This is where all the clients are connected to a centralised server, where all the information is stored
d) Topologies
(1) Bus
i) All clients have their own cable that connects to the Backbone.
ii) This backbone is a Bus, and allowss information to flow both ways to and from the server
iii) Terminators are required at each end to bounce packets
iv) Pros: easy to add to Cons: one point of failure for entire network
(2) Star
i) Each client has their own branch to connect to the client
ii) Pros: cable breakages only affect one client Cons: inefficient
(3) Ring
i) All clients are connected in a ring to eachother
ii) Pros: fast, can survive one breakage Cons: no central server
e) IP Address
(1) Every internet access point has an IP address, internal or external
(2) Internal IPs are used in LANs, and are often 192.168.x.x. These are not unique
(3) External IPs are WWW internet addresses. These are always unique
(4) IPv4 is the old standard, IPv6 is the new standard
f) MAC Address
(1) Every communications card has a hex MAC address, which is unique and canâ€™t be changed
(2) This is used to address internet or bluetooth etc
g) Packets
(1) Internet data is sent in packets, which are sections of code, mainly for web pages, that are sent from servers to a client using internet protocols
